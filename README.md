# Портфоліо проєктів — Христина Рябцева

Односторінковий сайт-портфоліо (`index.html`). Все в одному файлі, без бекенду.

## Як оновити цифри

Усі метрики вшиті в `index.html` вручну. Коли проєкти оновляться — цифри застаріють.

**Найпростіший спосіб:** відкрити Claude Code у папці VScode і написати:

> «Звір цифри у портфоліо з реальними проєктами і онови»

Claude перевірить версії у футерах проєктів, кількість комітів і виправить.

## Що де лежить (для ручного оновлення)

| Цифра | Де шукати актуальну | Де міняти в index.html |
|---|---|---|
| Версія GraficRobit (v0.5.93) | `projects/GraficRobit/RELEASES.md` (зверху) | пошук «v0.5» |

| Релізи MAD Backlog (29, v1.29.0) | футер mad_backlog/index.html | пошук «v1.29» |
| Автотести (178/178) | `GraficRobit/DEV_LOG.md` (останній запис) | пошук «178» |
| Типи змін (44) | код `graficrobit` репо: `src/lib/shifts.ts` | пошук «44» |
| Git-коміти (100+) | `git rev-list --count HEAD` у кожному проєкті | пошук «100+» |

## Правило версій

При **кожній** зміні сторінки підняти версію у футері (формат `v3.x.x`):
- виправлення тексту/багів → третя цифра (v3.8.1)
- нові блоки/функції → друга цифра (v3.9.0)

Цифри міняються в одному місці — у самому низу `index.html`, рядок `створено з Claude Code`.
